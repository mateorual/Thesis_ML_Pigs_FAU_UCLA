# Feature Selection

Treatment-specific feature selection for the thesis acoustic analysis of Yucatan mini-pig squeals after laryngeal surgery.

This implementation selects acoustic candidate features separately for the scar-only (`S`), unilateral COVR (`U`), and bilateral COVR (`B`) treatment groups. The selected features are intended for downstream longitudinal mixed-effects modeling, not as standalone inferential results.

## Method Summary

For each treatment group, the pipeline performs:

1. Data loading and schema validation.
2. Removal of near-zero-variance features.
3. Removal of highly correlated features using an unsupervised correlation filter.
4. Multinomial elastic-net stability selection across an `l1_ratio` grid.
5. Subject-stage-capped sensitivity analysis to reduce pig-dominated selections.
6. Final candidate-feature export for downstream LMM analysis.

The three-class learning task within each treatment group is:

```text
Pre vs Early Post vs Late Post
```

`Mid Post` recordings are excluded from the feature-selection task to focus on baseline, immediate post-surgical alteration, and late recovery.

## Expected Input

The main input is a consolidated acoustic-feature matrix generated by the feature-extraction implementation. It must be an Excel file with a sheet named `Features_Data` by default.

The first six columns must be metadata:

```text
Squeal, Subject, Treatment, Week, Num_Cycles, Stage
```

All remaining columns must be numeric acoustic features.

Valid values:

- `Treatment`: `S`, `U`, `B`
- `Stage`: `Pre`, `Early Post`, `Mid Post`, `Late Post`

## Run Feature Selection

If one consolidated input file contains all treatment groups:

```bash
python run_feature_selection.py ^
  --input path\to\Consolidated_Features_All_Groups.xlsx ^
  --sheet Features_Data ^
  --output-dir results\all_groups ^
  --groups S U B ^
  --B 500 ^
  --l1-ratios 0.3 0.5 0.7 0.9 ^
  --subject-stage-cap 15
```

If S/U and B are stored in separate input files, use separate runs:

```bash
python run_feature_selection.py ^
  --input path\to\Consolidated_Features_Groups_S_U_v1.xlsx ^
  --sheet Features_Data ^
  --output-dir results\S_U ^
  --groups S U ^
  --B 500
```

```bash
python run_feature_selection.py ^
  --input path\to\Consolidated_Features_Groups_B_v1.xlsx ^
  --sheet Features_Data ^
  --output-dir results\B ^
  --groups B ^
  --B 500
```

For a quick smoke test, reduce `--B`, for example:

```bash
python run_feature_selection.py --input path\to\features.xlsx --output-dir results\debug --groups S --B 5 --l1-ratios 0.5
```

## Main Outputs From `run_feature_selection.py`

For each processed group, the script writes:

```text
results/<GROUP>/final/final_candidate_features.csv
results/<GROUP>/final/features_for_LMM.csv
results/<GROUP>/stability_selection/stability_scores_combined.csv
results/<GROUP>/subject_capped/uncapped_vs_subject_capped_comparison.csv
results/<GROUP>/plots/
```

The file `features_for_LMM.csv` contains the final selected candidate features used for downstream longitudinal modeling.

## Summarize Selected Features Across S, U, and B

After running S/U and B feature selection, create overlap tables and figures:

```bash
python scripts\summarize_selected_features.py ^

  --results-b results\B ^
```

## Build Selected-Feature Datasets

The thesis used two selected-feature datasets:

- **Path 1:** union of selected features from `S + U + B`
- **Path 2:** union of selected features from `S + U`

To rebuild them from the consolidated feature matrix:

```bash
python scripts\build_selected_datasets.py ^
  --input path\to\Consolidated_Features_All_Groups.xlsx ^
  --sheet Features_Data ^
  --summary-dir outputs\selected_feature_summaries ^
  --output-dir outputs\selected_feature_datasets
```

Expected outputs:

```text
Selected_Features_Groups_S_U_B_v1.xlsx
Selected_Features_Groups_S_U_v1.xlsx
```

These files retain the metadata columns and only the selected acoustic-feature columns.

## Notes

- Feature selection is exploratory and treats individual squeals as independent observations.
- Final statistical interpretation should be based on downstream LMM analyses that account for repeated measurements and pig-level variability.
- The included selected-feature outputs correspond to the thesis analysis, where `S`, `U`, and `B` were selected independently and then combined into Path 1 and Path 2 datasets.
